### **(2) Detailed Description of the Mamdani Pipeline**

The slides break down the pipeline into three distinct phases to convert crisp data into a reasoned fuzzy conclusion.

#### **Step 1: Fuzzification**
The system takes a "crisp" input (a precise numerical value) and determines its **degree of membership** in various fuzzy sets using pre-defined membership functions.
*   **Example:** If an egg weighs **55 grams**, the fuzzification process checks this value against the "Small" and "Large" fuzzy sets.
*   **Output:** The crisp value is converted into fuzzy degrees, such as **small (0.75)** and **large (0.25)**.

#### **Step 2: Inference (Min-Max Composition)**
The system applies the fuzzy rule set to the fuzzified inputs. The slides specify that **Mamdani’s model** typically uses **Min-Max composition**.
*   **Rule Application:** The strength of the antecedent (the "IF" part) is applied to the consequent (the "THEN" part). 
    *   **Min() for AND:** If a rule has multiple conditions joined by "AND," the system takes the minimum degree of membership.
    *   **Implication:** For a rule like *"IF egg-size is small THEN boil-time is short,"* if the "small" membership is 0.75, the "short" output set is truncated at that 0.75 level.
*   **Aggregation (Max for OR):** Since multiple rules may fire simultaneously with different conclusions, the results are combined using an **OR relation**, taking the **Maximum** values across the overlapping output sets to form a final aggregated fuzzy shape.

#### **Step 3: Defuzzification**
The final aggregated fuzzy shape must be converted back into a single crisp numerical value that can be used for control (e.g., "boil for exactly 4.44 minutes"). The slides cover three primary methods for this:
1.  **MOM (Mean of Maximum):** Identifies the highest plateau of the aggregated shape and finds its horizontal midpoint.
2.  **COM (Center of Maximum):** Calculates a weighted average based on the central values of the contributing output sets.
3.  **COA (Center of Area / Centroid):** The most common method, which finds the "center of gravity" of the entire aggregated fuzzy area using summation or integration formulas.

### **Application Context**
As noted in **RS_Day2.pdf**, this pipeline is used in contemporary reasoning systems for **risk assessment in finance** and industrial tasks like **Plant Abnormal Situation Prediction**. In these hybrid systems, the fuzzy logic component handles imprecision (like "high pressure" or "soft clothes") to output precise control variables.
